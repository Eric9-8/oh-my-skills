# grok-search MCP

这是一个面向研究工作流的搜索型 MCP 示例，用于给 Agent 提供外部最新信息和可追溯证据。

## 它解决什么问题

当研究内容涉及下面这些信息时，只靠模型记忆往往不够稳：

- 最新法规与标准
- 新发布的论文、报告和博客
- 行业动态、融资、合作、产品发布
- 时间敏感的市场与公司信息

这时更合适的做法是让 Agent 在计划过程中判断“是否需要外部事实”，再按需调用搜索型 MCP。

## 在工作流里的位置

`用户问题 -> Agent Plan -> Skills 编排 -> 需要外部事实时调用 grok-search -> 综合与引用校验 -> 输出`

## 推荐用法

- 在 `question-refiner` 完成之后，针对时间敏感子问题触发
- 在 `got-controller` 拆出多个分支后，为需要最新事实的分支补充证据
- 在 `citation-validator` 发现高风险结论时，用于二次核验

## 不要把它当成什么

- 不要把它当成最终结论生成器
- 不要把“搜索到了”直接等同于“结论成立”
- 不要把真实密钥提交进 Git 仓库

## 脱敏配置模板

参见同目录下的 `.env.example`。

典型字段包括：

- `GROK_API_URL`
- `GROK_API_KEY`
- `GROK_MODEL`
- `TAVILY_ENABLED`
- `TAVILY_API_KEY`
- `FIRECRAWL_API_KEY`

## 验证建议

配置完成后，至少确认三件事：

1. 配置状态完整
2. 模型列表可拉取
3. 连接测试成功

## 分享建议

如果你要把这个 MCP 分享给同事，推荐只共享：

- 配置项说明
- 脱敏模板
- 使用场景
- 验证步骤

不要共享：

- 真实 API Key
- 本地日志目录
- 任何带有权限边界信息的私有配置
