# Case Study: 3DGS_ADAS_HiL_Tier1

这个案例用于说明：当研究目标足够复杂时，为什么要让 Agent 在 Plan 模式下自主编排 Skills 与 MCP，而不是手工逐个点工具。

## 原始问题

`Tier1 汽车供应商是否值得在 ADAS 硬件在环测试中引入 3DGS 神经仿真技术？`

这不是一个简单检索题，而是一个综合判断题，同时包含：

- 技术可行性
- 工程集成路径
- 成本与 ROI
- 法规窗口
- 行业竞争态势

## Agent 如何拆解

在这个项目里，可以看到一条清晰的拆解主线：

1. 先澄清研究对象、受众和交付物
2. 把主题拆成 `Why / How / Values / Cost / ROI`
3. 对多个维度并行研究
4. 对法规、论文、行业动态等时间敏感信息接入搜索型 MCP
5. 对多路结果做综合与引用校验
6. 最后输出成套交付物

## 产物链路

这个案例最终不是得到一段答案，而是得到一组交付物：

- `README.md`
- `full_report.md`
- `executive_summary.md`
- `presentation_outline.md`
- `sources/source_quality_table.md`

## 为什么这个案例适合分享

- 它能说明 Skills 不是工具清单，而是流程角色
- 它能说明 MCP 不是主角，而是外部能力接口
- 它能说明 Agent 的价值在于“先规划，再调用，再收口”
- 它能说明研究最终可以长成一条交付链，而不只是聊天记录

## 适合在会上怎么讲

推荐用一句话总结：

> 同一个研究目标，直接问模型通常只会得到一个答案；让 Agent 自主编排 Skills 与 MCP，得到的是一条完整交付链。
