# patent-architect

这是一个面向中文专利挖掘与专利材料生成的分享版 Skill。

## 适合什么场景

- 从现有项目里挖掘可申请的创新点
- 评估一个新方案是否具备专利价值
- 帮研发同学把技术实现转写成专利表达
- 在正式找代理人前，先形成技术交底草稿

## 它和普通研究 Skill 的区别

普通研究 Skill 关注“问题是否成立、结论是否可靠”，而 `patent-architect` 更关注：

- 技术问题是否明确
- 技术方案是否有结构化表达
- 技术效果是否能支撑创造性
- 创新点能否扩展成专利保护组合

## 推荐输入方式

### 1. 给项目路径

适合已有仓库，希望直接从代码和架构中挖创新点。

### 2. 给技术方案描述

适合已有构想，希望快速判断能否展开成专利。

### 3. 让 Skill 交互式引导

适合“有东西但说不清”的场景。

### 4. 明确说“帮我做专利挖掘”

适合只想先拿一份创新探索报告，不急着写正式专利材料。

## 推荐输出

- `innovation-exploration.md`
- `tech-disclosure.md`
- `patent-application-draft.md`

## 分享建议

如果你要把它讲给团队听，最适合强调三点：

1. 它不是法律代理替代品，而是研发侧的创新点整理器
2. 它先帮助确认“有没有可挖的点”，再决定“写成什么文档”
3. 它特别适合和代码分析、现有技术检索、图示生成一起使用
